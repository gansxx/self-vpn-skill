# Self VPN Skill

## 概览

本仓库提供一套可复用的 AI Agent 技能，帮助用户搭建个人专属 VPN 并完成客户端配置。目标是把“购买云主机 -> 安装服务 -> 导入客户端 -> 验证连通性”流程标准化，降低自部署门槛。

## 为什么要自部署 VPN

- 独立 IP，降低共享出口带来的连带风险  
  使用机场/共享 VPN 时，出口 IP 由多人共用。若同 IP 上存在滥用行为，可能影响你访问 AI 服务、触发风控或验证码频率升高。
- 性能和容量可控  
  你可以自行选择云厂商、地域、带宽与实例规格，不受第三方套餐的共享拥塞和流量策略影响。
- 隐私边界更清晰  
  自部署意味着你控制服务端和访问路径，减少把流量元数据交给第三方运营方的环节。
- 可维护性更高  
  节点不可用时可自行排查、迁移或重建，不依赖单一服务商的运营稳定性。

## 适用人群

- 有 AI 订阅，且希望长期稳定使用固定网络出口
- 对高峰期网络波动敏感，或每月流量消耗较大
- 对隐私和网络控制权有明确要求
- 具备基础终端操作能力，或愿意在 AI 引导下按步骤执行

## 仓库内容

- `skills/self-vpn-setup/`: 给 AI Agent 使用的技能目录
- `self-deploy.md`: 给人手动配置与排障的文档

## 快速开始

### AI 辅助配置

将下面提示词发给支持 skills 的 AI Agent（如 OpenClaw、Claude Code、Codex）：

```text
请从 GitHub 仓库 gansxx/self-vpn-skill 获取技能，并直接安装 skills/self-vpn-setup 到你的本地 skills 目录（例如 $CODEX_HOME/skills/self-vpn-setup）。安装后启用该技能，并按技能步骤引导我完成个人 VPN 部署与客户端配置。
```
- AI 会按技能流程逐步收集信息并引导部署。

### 手动配置

- 阅读 `self-deploy.md` 获取手动部署与排障流程。
- 适合具备基础 SSH / Linux / 云主机操作知识的用户。
