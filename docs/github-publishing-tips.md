# GitHub 发布技巧（本地总结）

## 1. 先做仓库首屏可读性

- README 首屏先回答三件事：是什么、适合谁、从哪里开始。
- 保持双入口：`AI 辅助部署` 与 `手动部署`。
- 用流程图和证据截图降低理解成本，而不是只堆长文本。

## 2. 元信息必须补全

- Description: 一句话说明核心价值。
- Topics: 至少覆盖产品类型、技术域、场景词。
- Social Preview: 上传 1280x640 封面图，提升外部分享点击率。

## 3. 图片与资源管理

- 所有展示资源放在 `assets/`，截图放 `assets/screenshots/`。
- 文件名用语义化命名，如 `client-latency-test.png`。
- 本地敏感图（如私域二维码）写入 `.gitignore`，避免误上传。

## 4. 提交策略

- 一次提交只做一类改动：文档、资产、流程、脚本分开。
- 提交信息格式建议：`docs: ...`、`feat: ...`、`fix: ...`。
- 推送前执行 `git status`，确认没有意外文件。

## 5. 发布前自检清单

- README 内所有本地链接可打开。
- 关键流程与截图一致，不出现“文档说 A、截图是 B”。
- 新用户 30 秒内能找到安装入口与排障入口。
- Issues 与直接联系方式都在首页可见。
